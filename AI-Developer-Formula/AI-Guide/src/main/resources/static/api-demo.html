GET /users/masked/{id} Conditional masking — MaskMeOnInput condition

Fields with @MaskMe(conditions = {MaskMeOnInput.class}) are masked only when the Mask-Input header equals "maskMe". Try different values to see the difference.

Show raw JSON
GET /users/user/{id} Domain entity — AlwaysMaskMeCondition applied

Returns the User domain entity with AlwaysMaskMeCondition applied. Fields like email, password, birthDate, and balance are always masked. The address is excluded via @ExcludeMaskMe.

Show raw JSON
GET /users All users — MaskMeOnInput + PhoneMaskingCondition

Demonstrates multiple conditions on the same request. MaskMeOnInput masks fields when header matches, and PhoneMaskingCondition masks only the specific phone number provided in the Mask-Phone header.
