Implementation Details

  1. Each agent (e.g., TicketAnalysisAgent):
    • Loads its prompt from the corresponding markdown file in resources/agents/.
    • Uses an LLM client interface to process input and return output.
    • Is stateless for easy testing and scaling.
  2. Orchestrator (WorkflowOrchestrator):
    • Coordinates agent execution, manages a shared context, and supports human-in-the-loop checkpoints.
    • Should be easy to refactor into a Spring @Service or @Component.
  3. Context (WorkflowContext):
    • Holds all shared data/state for a workflow run.
    • Is passed between agents.
  4. LLM Abstraction:
    • LlmClient is an interface for LLM operations.
    • Implementations: ClaudeApiClient, OpenAiApiClient, etc.
    • The orchestrator and agents depend on LlmClient, not a specific LLM.
    • It should be easy to add new LLMs (e.g., Gemini) by implementing this interface.
  5. Markdown Loader Utility:
    • MarkdownLoader loads and fills markdown templates from resources.
  6. Testing:
    • Place unit tests under src/test/java/com/javamsdt/aidevworkflow/.
  7. Spring-Ready:
    • The structure and interface-driven design should make it easy to add Spring Boot later and use dependency injection.
  8. Human-in-the-Loop:
    • Insert checkpoints in the orchestrator for human confirmation (e.g., via CLI, web UI, or notification).
    • Example: After project setup, before deployment.

Sample Agent
package com.javamsdt.aidevworkflow.agents;

import com.javamsdt.aidevworkflow.llm.LlmClient;

public class TicketAnalysisAgent {
    private final LlmClient llmClient;

    public TicketAnalysisAgent(LlmClient llmClient) {
        this.llmClient = llmClient;
    }

    public String analyze(String ticketText) {
        String prompt = MarkdownLoader.load("agents/ticket_analysis.md").replace("{{ticket}}", ticketText);
        return llmClient.completePrompt(prompt);
    }
}
