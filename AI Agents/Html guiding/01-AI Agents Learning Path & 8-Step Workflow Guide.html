1. AI Agents: The Basics

What is an AI Agent?

  • Definition: An AI agent is a software entity that perceives its environment, reasons about it, and acts autonomously to achieve goals.
  • Key Features: Perception, Reasoning, Action, Autonomy.
  • Types: Reactive, Deliberative, Collaborative, Hybrid.
Are AI agents language-specific?
No! The concept is universal. You can implement agents in Java, Python, or even orchestrate them using markdown files, depending on the platform.

How Do Agents Work?

  • Some use rules (if-then logic).
  • Modern agents often use machine learning or large language models (LLMs) like Claude or GPT.
  • Agents can be orchestrated via code (Java, Python) or via markdown instructions (platform-specific).

3. Deep Dive: Claude, Java & Markdown Agents

Claude’s Agent System

  • Claude allows you to define agents using markdown files that describe the agent’s purpose, instructions, and workflow.
  • These markdown files are not language-specific and can be edited by anyone familiar with the workflow.
  • The Claude platform reads these markdowns and uses them as prompts or instructions for the LLM.
# Agent: Ticket Analysis ## Purpose Summarize and analyze incoming tickets for key requirements and ambiguities. ## Instructions - Read the ticket details provided. - Extract main requirements. - Identify unclear or missing information. - Suggest clarifying questions. ## Input - Ticket text ## Output - Summary - List of requirements - List of clarifying questions

How Java Integrates with Claude Agents

  • Java code acts as the orchestrator: it loads the markdown agent files, fills in variables, and sends them to Claude’s API.
  • The response from Claude is parsed and used in your workflow.
  • This hybrid approach lets you keep business logic in Java and agent instructions in markdown.
Tip: Explore the agent folder in your Claude project to see real examples of markdown agent definitions.

5. Step-by-Step Guide: Implementing the Workflow in Java

Step 1: Set Up Your Java Project

  • Install IntelliJ IDEA and create a new Java project.
  • Add dependencies for HTTP requests (e.g., OkHttp or Apache HttpClient).
  • Organize your project with a package for each agent (e.g., agents.ticket, agents.setup).

Step 2: Load and Use Markdown Agent Files

  • Place your markdown agent files in a resource folder (e.g., src/main/resources/agents/).
  • Write a Java utility to read and fill in variables in these markdown files.
// Example: Load markdown agent file String agentMarkdown = Files.readString(Paths.get("src/main/resources/agents/ticket_analysis.md")); String prompt = agentMarkdown.replace("{{ticket_text}}", ticketText);

Step 3: Connect to Claude API

  • Sign up for Claude API and get your API key.
  • Use Java HTTP client to send the prompt to Claude and receive the response.
// Example: Send prompt to Claude API (pseudo-code) HttpRequest request = HttpRequest.newBuilder() .uri(URI.create("https://api.anthropic.com/v1/complete")) .header("Authorization", "Bearer YOUR_API_KEY") .POST(HttpRequest.BodyPublishers.ofString("{ \"prompt\": \"" + prompt + "\" }")) .build();

Step 4: Implement Each Agent as a Java Class

  • Create a class for each agent (e.g., TicketAnalysisAgent).
  • Each class loads its markdown, sends it to Claude, and processes the response.
public class TicketAnalysisAgent { public TicketAnalysisResult analyze(String ticketText) { String prompt = loadAndFillMarkdown("ticket_analysis.md", ticketText); String response = sendToClaude(prompt); return parseTicketAnalysis(response); } }

Step 5: Orchestrate the Workflow

  • Create a main orchestrator class that calls each agent in sequence, passing results from one to the next.
public class AIWorkflowOrchestrator { public static void main(String[] args) { TicketAnalysisAgent ticketAgent = new TicketAnalysisAgent(); ProjectSetupAgent setupAgent = new ProjectSetupAgent(); // ... other agents TicketAnalysisResult ticket = ticketAgent.analyze("ticket details"); Project project = setupAgent.setup(ticket); // ... continue workflow } }

Step 6: Test and Iterate

  • Test each agent individually and as part of the full workflow.
  • Refine your markdown agent instructions for better results.
  • Log and handle errors gracefully.

Step 7: Expand and Customize

  • Add more agents or steps as needed.
  • Customize markdown files for different projects or workflows.
  • Consider adding a simple UI or CLI for easier use.
